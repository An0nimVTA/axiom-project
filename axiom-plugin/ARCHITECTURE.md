# Новая архитектура AXIOM Geopolitical Engine

## Обзор

Проект переходит на чистую многослойную архитектуру для улучшения поддержки и расширяемости.

## Структура пакетов

```
axiom-plugin/
├── src/main/java/com/axiom/
│   ├── controller/          # Обработка команд и событий
│   ├── service/             # Бизнес-логика
│   ├── model/               # Модели данных
│   ├── repository/          # Работа с хранилищами
│   ├── util/                # Утилиты и хелперы
│   ├── config/              # Конфигурация
│   ├── exception/           # Кастомные исключения
│   └── AxiomApplication.java # Главный класс (будущий)
```

## Принципы

1. **Single Responsibility**: Каждый класс имеет одну ответственность
2. **Dependency Injection**: Зависимости передаются через конструкторы
3. **Interface Segregation**: Интерфейсы должны быть небольшими и специфичными
4. **Testability**: Код должен быть легко тестируемым

## Миграционный план

1. Перенос моделей данных в `com.axiom.model`
2. Создание репозиториев в `com.axiom.repository`
3. Рефакторинг сервисов в `com.axiom.service`
4. Создание контроллеров в `com.axiom.controller`
5. Обновление главного класса

## Конвенции

- Все сервисы должны иметь интерфейс и реализацию
- Репозитории должны работать с интерфейсами моделей
- Контроллеры должны быть тонкими и делегировать логику сервисам
